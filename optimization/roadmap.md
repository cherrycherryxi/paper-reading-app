## 2. 三层目标

**长期(6 个月,北极星):**
「每次读纸质书,从拍照摘抄到事后回顾,这个 app 是我不假思索的默认工具。」
可观测代理指标(每周日记一次,3 个数):本周使用天数 / 本周新增摘抄数 / 本周回顾·检索·关联操作次数。第三个数长期为 0 = 产品只做对了一半(只采集、无回流)。

**中期(8 周,主题制——每 4 周一个主题,不是功能清单):**
- **Theme 1「采集顺滑」(~7/10 前)**:真机环境下拍照→OCR→成卡的全链路可靠、快、不丢数据。验收:连续 2 周真实使用零孤儿卡、零丢失、零「等太久放弃」。
- **Theme 2「回顾有价值」(~8/7 前)**:让积累的摘抄回流到阅读生活——搜索、关联、某种形式的回顾(随机重现/按书回顾/导出分享卡皆可,届时由信号决定)。验收:北极星第三个数 > 0 且连续 2 周增长。

**短期(本周 + 下周):**
1. 建立 `optimization/signals.md` 使用信号日志(见 §3 用户研究员)。
2. 真机复测 OPT-042/043(导入护栏 + OCR 孤儿卡)——这是 Theme 1 的起点验收。
3. 给夜间流水线装「预算阀」(见 §5),防止修缮永动机。
4. backlog 里 6 个 new + 3 个 triaged:按「对北极星的贡献」重排一次,答不上来的降 P3。

## 3. 角色体系(轻量,不加新 cron)

现有夜间 3 agent = **工程维护部**,保留但限额(§5)。新增 4 个角色全部由白天交互 session 扮演,靠**固定仪式**而非新 agent 落地:

| 角色 | 节奏 | 做什么 | 产出物 |
|---|---|---|---|
| **产品负责人**(Claude 扮演) | 每周一 ≤15 min | 读 signals.md + 上周 PR,定本周**唯一焦点**,砍跑题项 | roadmap.md 短期节更新 |
| **用户研究员**(= 你本人) | 每次真实使用后 ≤1 min | 摩擦/惊喜随手记一行,不分析 | `optimization/signals.md` |
| **架构守门人**(Claude 扮演) | 双周一次 ≤30 min | 盯结构性债。规则先行:`app_server.py`(5485 行)**超 6500 行或单函数超 150 行才拆**,否则不动 | 必要时出拆分 OPT |
| **发布 QA**(你+Claude) | 每个功能 PR 合并前 | 3–5 条真机验收清单(iPhone 12),走完才合 | PR 评论里的清单 |

关键设计:**用户研究员是唯一不能外包给 AI 的角色**。signals.md 一行格式:`日期 | 场景 | 摩擦或惊喜(一句话)`。它是产品负责人和夜间 triage 的共同输入——今后 triage 优先提拔有 signal 佐证的项。

## 4. 迭代节奏(适配每天 1–2h)

```
周一   15min 产品负责人仪式(定唯一焦点)→ 剩余时间开干
周二–四 每天 1–2h:只做本周焦点(功能轨),WIP = 1
周五   收尾 + 合并 + 真机验收(发布 QA)
周末   真实使用(读书!)+ 顺手记 signals;可选 30min 处理夜间 PR 积压
夜间   agent 跑修缮轨(限额内)
```

- 白天时间是稀缺资源,**只花在功能轨**;审夜间修缮 PR 每天 ≤20 min,超了就攒到周末批处理。
- 1–2h 做不完一个功能很正常:按「每天结束时分支可合并或可搁置」切片,不留半成品过夜污染主干。

## 5. 速度/质量平衡规则

1. **双轨制**:功能轨(白天,你主导)允许「粗糙先用」——UI 类可以不写测试,真机用一周再补;修缮轨(夜间 agent)维持现状的强测试纪律。
2. **质量闸门分级**:涉及**数据安全**(import/export、state 写入、DB schema、OCR 落盘)= 必须测试 + 真机双验,无例外(bug-274 的教训);纯 UI/文案 = 真机即可,test-on-regression。
3. **修缮预算阀**:夜间 implement agent 每周 ≤ 4 个 PR;explore 蓄水池只进不出会无限膨胀(已 69 条),每月产品负责人仪式顺手 prune 一次,90 天无人提拔的条目归档。
4. **北极星税**:今后每个新 OPT 登记时多写一行 `northstar: <对北极星的贡献,一句话>`,写不出来自动 P3。这是防「工程上正确但产品上无谓」的最便宜机制。
5. **止损规则**:任何功能真机用 2 周后 signals 里零正反馈 → 下个周一仪式上讨论砍掉,沉没成本不进决策。
