# Paper Reading App — 需求文档 v1.0

> 本文档供 AI 编程工具（Cursor / Copilot 等）直接使用，按优先级排列，每条需求包含背景、目标行为和验收标准。

---

## 项目背景

这是一个面向手机网页（优先适配 iPhone 12）的纸质书阅读记录工具。技术架构为：

- **前端**：`index.html` + `app.js` + `chat.js` + `styles.css`，运行在本地静态服务器（端口 4173）
- **后端**：`log_server.py`（Python），运行在端口 8787，使用 SQLite 存储数据
- **外部 API**：DeepSeek API，用于 OCR 识别和聊天功能
- **账号系统**：后端管理，登录 token 保存在 `localStorage`

## P0 — 功能性 Bug（需优先修复）

### Bug 1：OCR 识别无结果

**现象**：用户上传图片后，`/api/ocr` 接口调用成功（无网络报错），但前端收不到识别文本，界面无任何内容显示。

**排查与修复要求**：

1. 检查 `log_server.py` 中 `/api/ocr` 路由的实现，确认向 DeepSeek API 发送请求时：
   - 图片是否以正确格式传递（base64 编码 + 正确 `media_type`）
   - 使用的模型是否支持视觉输入（需使用支持图像的多模态模型，如 `deepseek-vl` 系列）
   - 请求体结构是否符合 DeepSeek Vision API 规范
2. 检查后端是否正确解析并返回 OCR 结果，错误不能被静默吞掉，需写入日志
3. 检查前端 `app.js` 中接收 OCR 响应后的处理逻辑，确认结果被正确渲染到界面上

**验收标准**：用户上传一张包含文字的书页照片后，界面能在合理时间内（≤ 10 秒）展示识别出的文本内容。

---

### Bug 2：书单顺序 / 分类混乱

**现象**：当用户书单中有多本书时，书籍的显示顺序不固定，或分类归属不正确。

**排查与修复要求**：

1. 检查后端 `/api/state`（GET）返回书单数据时，是否包含稳定的排序字段（如 `created_at` 时间戳或用户自定义 `order`）
2. 检查前端渲染书单列表时，是否对数据进行了明确排序（如按添加时间升序），而非依赖后端返回的随机顺序
3. 检查书籍分类字段在 `PUT /api/state` 保存和后续 GET 读取时是否完整一致，排查是否存在字段名拼写不一致或 JSON 序列化丢失的问题

**验收标准**：书单列表按添加时间稳定排序；分类标签在页面刷新或重新登录后保持正确，不发生错位或丢失。

## P1 — 新功能（搜索）

### 功能：全局搜索

**背景**：页面上已有搜索栏 UI，但目前没有任何搜索逻辑与之绑定。

**需求描述**：实现支持以下三个维度的搜索功能：

| 维度 | 搜索范围 |
|------|----------|
| 书名 | 书单中每本书的标题字段 |
| 作者 | 书单中每本书的作者字段 |
| 摘抄内容 | 用户保存的所有摘抄文本 |

**实现方式（推荐前端过滤，适合当前数据规模）**：

1. 在页面初始化时，确保书单和摘抄数据已完整加载到前端内存
2. 搜索栏输入时触发实时过滤（`input` 事件，无需按回车），对书名、作者、摘抄内容做大小写不敏感的模糊匹配
3. 搜索结果分区展示：匹配到书名/作者的显示为"书籍"结果，匹配到摘抄内容的显示为"摘抄"结果
4. 搜索框清空时恢复默认视图

**验收标准**：输入任意关键词后，0.3 秒内页面实时展示匹配结果；搜索结果覆盖书名、作者、摘抄三个维度；无匹配时显示空状态提示。

## P2 — UI 风格重设计

### 设计方向

参考风格如下图（ChaTin App UI），核心特征为：

- **底色**：奶油米白（非纯白，推荐 `#F5F0E8` 或 `#FAFAF5`），而非当前的普通白色或灰色
- **主色调**：纯黑文字 + 亮黄色（`#FFD700` 或 `#F5C518` 系列）作为主 CTA 按钮和强调色
- **辅助色**：少量彩色卡片（淡粉、淡绿）用于区分内容模块，整体不超过 3 个辅助色
- **字体**：大号无衬线粗体（bold/black weight）用于标题，正文使用中等字重，保持层次感
- **卡片**：明显圆角（`border-radius: 16px~24px`），白色卡片浮在米白底色上，带轻微投影
- **按钮**：主按钮为亮黄色椭圆形，内含右箭头图标；次级操作用黑色圆形图标按钮
- **标签/Chips**：黑色填充圆角标签，白色文字，用于分类和历史记录展示
- **装饰元素**：可在书单封面或空状态页加入手绘风小插图/贴纸，增加活泼感
- **聊天界面**：深色（近黑）背景气泡用于 AI 回复，白色气泡用于用户消息

### 改造范围

需对以下文件进行样式重构：

- `styles.css`：全局重写配色变量、字体、间距、圆角、阴影等基础 token
- `index.html`：检查并调整 class 命名，确保与新样式对应
- `app.js` / `chat.js`：如有内联样式或动态注入的 class，一并更新

### 验收标准

在 iPhone 12 Safari 浏览器中打开，整体视觉风格符合上述参考图的调性：奶油底色可见、主 CTA 按钮为亮黄色、标题层次清晰、卡片圆角明显；不出现纯白背景、蓝色默认链接色或无样式的原生控件。

## 附：现有技术约束提示

- 前端通过 `window.PAPER_READING_APP_CONFIG.backendBaseUrl` 配置后端地址，修改任何接口路径需同步前后端
- iPhone 访问时 `backendBaseUrl` 必须为局域网 IP，不能用 `127.0.0.1`
- 登录 token 存于 `localStorage`，其余数据均由后端主存，前端改动不能破坏现有会话逻辑
- 后端日志可通过 `http://<IP>:8787/debug/logs` 快速查看，排查 Bug 时优先检查此页面
